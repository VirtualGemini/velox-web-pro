# ip2region xdb data

This directory contains the official ip2region xdb data used by log IP
location resolving.

- `ip2region_v4.xdb`
- `ip2region_v6.xdb`

The default backend configuration loads both files from:

- `classpath:ip2region/ip2region_v4.xdb`
- `classpath:ip2region/ip2region_v6.xdb`

Override the paths via configuration if you ship the data elsewhere:

```yaml
system:
  log:
    ip-location:
      xdb-v4-path: classpath:ip2region/ip2region_v4.xdb
      xdb-v6-path: classpath:ip2region/ip2region_v6.xdb
```

These data files are not generated by the Maven dependency. If the configured
files are missing, public IP locations fall back to `unknown`; intranet
addresses still fall back to `intranet`.

The `*_source.txt` files are the raw generation sources for the xdb databases.
They are not used at runtime and must not be packaged into the jar.
